# Run 20260324_173812 问题总结（research_writing）

## 主要问题

1. **检索词生成错误**：research step 直接把内部 goal 文本当成 query，导致搜索词不可用。  
2. **相关性校验缺失**：返回了与显卡无关的来源也被当成成功。  
3. **缺少重试/改写机制**：低质量结果没有触发自动改写 query 后重搜。  
4. **证据闸门缺失**：research 失败后仍继续生成 final report。  
5. **置信度失真**：执行成功被误当成事实可靠，confidence 偏高。  
6. **风险信号未落地**：异常情况未记录可用于“自动刹车”的 risk signals。  
7. **状态机不一致**：步骤完成后 phase 未推进到 completed。  
8. **任务拆解过粗**：未拆分 benchmark 与价格/性价比研究维度。

## 对应修复方向

- query 构造：把 task/goal 归一化为用户检索词候选，避免 instruction 文本直传。  
- relevance gate：基于任务词覆盖和来源数做质量判定，不达标则暂停。  
- retry loop：多候选 query 自动重试。  
- evidence gate：final_report 前必须存在“已验证 research 结果”。  
- risk signals：把低来源数、不相关来源、query 异常写入结构化字段。  
- phase 修复：执行阶段完成后更新 current_phase 为 completed。  
- planner 拆解：research_writing 增加 benchmark 与价格两个研究子步。
